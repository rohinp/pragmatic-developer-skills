# Source Map

This skill was initialized from PDFs that were available in the parent workspace. The PDFs are source material only; they do not need to be part of the skill package. The notes below intentionally paraphrase durable practices and outline topics instead of copying book text.

## Domain-Driven Design Topics

- Knowledge crunching and continuous learning.
- Ubiquitous language across speech, tests, code, diagrams, and documents.
- Binding model and implementation.
- Layered architecture and isolating the domain.
- Entities, value objects, services, modules, aggregates, factories, and repositories.
- Supple design: intention-revealing interfaces, side-effect-free functions, assertions, conceptual contours, standalone classes, closure of operations, and declarative design.
- Refactoring toward deeper insight.
- Strategic design: bounded context, continuous integration, context map, shared kernel, customer/supplier, conformist, anticorruption layer, separate ways, open host service, and published language.
- Distillation: core domain, generic subdomains, domain vision, highlighted core, segregated core, abstract core, and choosing refactoring targets.

## Test-Driven Development Topics

- Red-green-refactor.
- Money example and xUnit example as demonstrations of incremental design.
- TDD patterns, red bar patterns, testing patterns, green bar patterns, xUnit patterns, design patterns, refactoring, and mastering TDD.
- Design pressure from tests: equality, immutability, value objects, side effects, privacy, factory methods, abstraction, and mixed cases.
- Quick green tactics: fake implementation, obvious implementation, and triangulation.

## Refactoring Topics

- Refactoring as behavior-preserving design improvement.
- Code smells as prompts for investigation.
- Small named transformations such as extract, inline, move, rename, replace conditionals, introduce parameter object, preserve whole object, replace primitives with objects, and simplify method calls.
- Tests as safety net for restructuring.

## Design Patterns Topics

- Shared pattern vocabulary and experience reuse.
- Strategy, Observer, Decorator, Factory Method, Abstract Factory, Singleton, Command, Adapter, Facade, Template Method, Iterator, Composite, State, Proxy, Compound Patterns, and MVC.
- Core principles: encapsulate what varies, favor composition over inheritance, program to interfaces, strive for loose coupling, open-closed principle, dependency inversion, least knowledge, and single responsibility.
- Pattern use in real systems, including avoiding misuse and over-application.

## Pragmatic Programmer Topics

- Pragmatic philosophy: responsibility, entropy, good-enough software, knowledge portfolio, communication.
- Pragmatic approach: good design, DRY, orthogonality, reversibility, tracer bullets, prototypes, domain languages, estimating.
- Basic tools: plain text, shell, editor fluency, version control, debugging, text manipulation, engineering daybooks.
- Pragmatic paranoia: contracts, failing fast, assertions, resource balance, not outrunning what is known.
- Bend or break: decoupling, real-world events, transformations, inheritance tradeoffs, configuration.
- Concurrency: temporal coupling, shared state, actors/processes, blackboards.
- Coding topics: instinct, programming by coincidence, algorithm speed, refactoring, test to code, property-based testing, safety, naming.
- Projects: requirements, collaboration, agility, pragmatic teams, starter kits, user delight, and professional pride.
